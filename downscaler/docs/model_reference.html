Architecture: conv U-Net, transformer bottleneck, EDM wrapper

Three nested pieces: a convolutional encoder/decoder, a stack of windowed-attention transformer blocks where the grid is smallest, and the EDM preconditioning wrapper that turns the network into a denoiser Dθ.

The transformer blocks (model/attention.py)

Each of the attn_depth = 4 blocks is a pre-norm transformer: windowed multi-head self-attention, then an MLP, each with a residual connection. Specifics:

  • Windowed attention (Swin-style). Tokens attend within 16×16 windows; blocks alternate unshifted / cyclically-shifted windows so information crosses window boundaries every second block. A learned relative position bias per head gives the attention directional (north–south vs east–west) structure. Non-divisible grids are padded and the padded tokens are masked out of attention.
  • adaLN-Zero noise conditioning. The EDM noise embedding modulates the block's two LayerNorms (scale, shift) and gates both residual branches. Gates are zero-initialized, so a fresh block is an exact identity — turning attention on does not perturb early training; the network opens the gates as it learns to use them.
  • Memory. Blocks are activation-checkpointed like the ResBlocks, and the attention runs only at ÷8 resolution, so the full-resolution single-GPU memory budget is preserved.

ResBlocks and noise conditioning

Every ResBlock is Conv → AdaGN → SiLU, twice, plus a residual. AdaGN is GroupNorm whose scale/shift come from the noise embedding — a sinusoidal embedding of c_noise = ln(σ)/4 passed through a 2-layer MLP (emb_dim = 256). Downsampling is a stride-2 conv; upsampling is bilinear + conv.

EDM preconditioning (the wrapper)

The U-Net's raw output Fθ is blended with the noisy input using σ-dependent coefficients (c_skip, c_out, c_in — Karras et al. 2022, σdata=1): Dθ(x, σ) = c_skip·x + c_out·Fθ(c_in·x, c_noise). Conditioning channels are concatenated un-scaled. Inference is a single-shot conditional sample via the Heun ODE sampler — 18 steps, 35 network evaluations — not an autoregressive rollout.

Two preprocessing rules that are load-bearing

Precip transform. ERA5 tp is a 12 h accumulation in metres; the HRRR target is a 6 h accumulation of ln(mm + 1e-5). Channel 14 goes through log_precip_m_to_mm (m→mm, same log, same dry-pixel floor of −11.51) before regridding. Without it the channel is 1000× off scale and 77% zeros. The 12 h→6 h window mismatch remains — the model learns the disaggregation, with specific humidity as the clean (window-free) moisture predictor.

Wind rotation. HRRR u10/v10 are grid-relative; ERA5's are Earth-relative (up to ±22.8° apart at the CONUS edges). The dataset rotates the HRRR wind targets to Earth-relative before normalizing, so model outputs are directly comparable to ERA5/GenCast. New wind channels must be declared in data.hrrr_wind_pairs or they are silently left unrotated.

All outputs are the HRRR full-field state at the conditioning time (not a tendency), in per-channel z-score space; σdata=1 is valid because the stats make targets ≈ unit variance. One sample = one timestamp; ensemble members come from re-sampling the initial noise.

The channel-count contract

train.py asserts this at startup and refuses to run on mismatch — growing the variable lists is a config edit, made in configs/data/era5_hrrr.yaml and configs/model/unet.yaml together:

n_era5_cond  == len(era5_variables)                        # 9
n_prognostic == len(hrrr_prognostic_variables)             # 4
n_diagnostic == len(hrrr_diagnostic_variables)             # 1
model.n_output_ch == n_prognostic + n_diagnostic           # 5
model.n_input_ch  == n_output_ch + n_era5_cond + n_static  # 5 + 9 + 2 = 16
